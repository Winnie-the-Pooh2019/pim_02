# Exercise 03 — Glossary (Task 1, prefix: bsh)

Дата добавления: PIM_02

Ниже — таблица ключевых понятий и терминов, встречающихся в задаче 1 (Запись на стрижку). Колонки: термин (EN), термин (RU), определение, примечания.

| Term (EN)            | Term (RU)                       | Definition                                                                                       | Notes                                                      |
|----------------------|---------------------------------|--------------------------------------------------------------------------------------------------|------------------------------------------------------------|
| Appointment          | Запись                          | Договорённость о посещении салона в конкретное время, оформляемая пользователем через систему.   | Может быть для зарегистрированного пользователя или гостя. |
| Registered user      | Зарегистрированный посетитель   | Пользователь с учётной записью: хранится история записей, предпочтения, контакты.                | Позволяет персонализацию и быстрый повторный заказ.        |
| Guest                | Незарегистрированный посетитель | Пользователь, делающий разовую запись без аккаунта; минимальные данные.                          | Упрощённый поток, может требовать проверочного кода.       |
| Service              | Услуга                          | Тип процедуры (стрижка, укладка и т.п.) с длительностью и ценой.                                 | Может иметь дополнительные опции.                          |
| Master               | Мастер                          | Сотрудник, выполняющий услугу; имеет профиль, расписание и отзывы.                               | Привязка к слотам расписания.                              |
| Manager              | Менеджер                        | Сотрудник, вводящий/корректирующий расписание, связывающийся с клиентами, отмечающий выполнение. | Может вести отчёты и ручные операции.                      |
| Admin                | Администратор системы           | Роль, управляющая пользователями, правами и параметрами системы.                                 | Доступ к журналам, бэкапам и настройкам.                   |
| Schedule             | Расписание                      | Набор временных слотов мастера, доступных для записи.                                            | Может учитывать перерывы и санитарные блокировки.          |
| Slot                 | Интервал / Слот                 | Конкретный временной отрезок в расписании, со статусом (свободен/забронирован/занят).            | Единица бронирования.                                      |
| Reminder             | Напоминание                     | Автоматическое уведомление клиента о предстоящей записи.                                         | Каналы: SMS, push, мессенджеры, email.                     |
| Channel              | Канал связи                     | Средство отправки уведомлений (SMS, Telegram, WhatsApp, VK, push, email).                        | Конфигурируемо по пользователю.                            |
| Sanitation           | Санитарная обработка            | Процедура обработки помещений и инструментов; фиксируется в системе.                             | Требование регуляторов, влияет на доступность слотов.      |
| NPS                  | Net Promoter Score              | Метрика лояльности, собираемая после оказания услуги (оценка/отзыв).                             | Важна для владельца сети.                                  |
| Average check        | Средний чек                     | Средняя сумма, потраченная клиентом за визит.                                                    | KPI для бизнеса.                                           |
| Cancellation         | Отмена записи                   | Действие по отмене ранее созданной записи; правила зависят от политики.                          | Может быть платной/бесплатной.                             |
| No-show              | Неявка                          | Клиент не пришёл на забронированную услугу.                                                      | Влияет на KPI и может требовать штрафов.                   |
| Payment              | Оплата                          | Процесс приёма средств за услугу (онлайн/офлайн).                                                | Требует интеграции с платежными провайдерами.              |
| Personal data / GDPR | Персональные данные             | Контактная и личная информация клиента; требует защиты и соответствия законам.                   | Хранение и обработка по правилам безопасности.             |
| User story           | Пользовательская история        | Описание требования с точки зрения пользователя: цель, мотивация, критерии приёма.               | Полезно для разработки и тестирования.                     |

Примечания:

- Таблица даёт компактный машинно-читаемый обзор терминов; при необходимости можно расширить каждую строку ссылкой на спецификации или пример JSON-поля.
