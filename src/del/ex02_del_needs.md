# Exercise 02 — Interests, needs, problems of stakeholders

Контекст: задача 2 — "Доставка заказов". Ниже приведён анализ интересов, потребностей и проблем ключевых стейкхолдеров, взятых из `src/del/ex00_del_stakeholders.md`.

Дата добавления: 10.10.2025

|   Идент. | Стейкхолдер                               | Интересы (что выгодно)                                                            | Потребности (что нужно от системы)                                                           | Проблемы (что мешает достижению)                                                         |
|---------:|-------------------------------------------|-----------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------|
| st002001 | Владелец(и) стартапа / Учредители         | Рост выручки, окупаемость, устойчивость бизнеса                                   | Быстрый выход на рынок, прозрачность метрик (заказы, доходы), низкие операц. расходы         | Небольшой бюджет, неопределённость спроса, риск плохой операционной реализации           |
| st002002 | Оператор ввода заказов                    | Эффективность ввода, минимизация ошибок                                           | Удобный интерфейс ввода, валидация данных, шаблоны импорта, лог операций                     | Неполные/разнородные данные от магазинов, ручная работа, высокая нагрузка при пиках      |
| st002003 | Курьер                                    | Стабильный поток заказов, понятные маршруты, своевременная оплата                 | Мобильное приложение с актуальными заказами, навигацией, статусами и начислениями            | Неправильная/неактуальная информация о заказе, плохо продуманные тайминги, слабая оплата |
| st002004 | Диспетчер                                 | Контроль выполнения, минимизация простоев курьеров, своевременная реакция на сбои | Панель мониторинга курьеров/заказов, возможность перераспределять заказы                     | Большое количество изменений вручную, плохая видимость статусов, задержки в обновлениях  |
| st002005 | Магазин / Предприятие питания (поставщик) | Быстрая и точная передача заказов, минимальные ошибки комплектации                | Чёткий формат заказов/API/инструкции, подтверждения при приёме заказа                        | Различные форматы данных, требования к упаковке/времени, человеческие ошибки при вводе   |
| st002006 | Клиент (получатель заказа)                | Быстрая и точная доставка, сохранность товара, прозрачные статусы и возвраты      | Удобный интерфейс для заказа/отслеживания, уведомления о статусе, простая связь с поддержкой | Опоздания, повреждения, неполная информация о доставке, неудобные окна доставки          |
| st002007 | Бухгалтерия (внешняя система)             | Корректные данные для расчёта выплат и расчётов с поставщиками                    | Структурированный экспорт/интеграция данных (API/файлы), согласованные форматы               | Несогласованные форматы данных, задержки с передачей данных, ручная очистка данных       |
| st002008 | Администратор системы                     | Стабильность платформы, безопасность, управление доступами                        | Удобная админ-панель, журналы действий, управление ролями, мониторинг и бэкапы               | Сложные процедуры управления, уязвимости, нехватка инструментов для аудита               |
| st002009 | Команда разработки / DevOps               | Быстрое развёртывание фич, качество кода, стабильные среды                        | Чёткие требования, тестовые данные, процессы CI/CD, доступ к логам и метрикам                | Неоднозначные требования, нестабильные окружения, недостаток telemetriy/логов            |
| st002010 | Провайдер платежей (payment gateway)      | Надёжные транзакции, минимальные риски фродa                                      | Безопасные API, webhooks, понятная обработка статусов платежей                               | Отозванные платежи, интеграционные задержки, различия в статусах платежей                |
| st002011 | Служба поддержки клиентов                 | Быстрая обработка обращений, высокий уровень удовлетворённости клиентов           | Дашборд с информацией по заказам, история взаимодействий, инструменты эскалации              | Нехватка информации для решения вопросов, большое число простых повторяющихся запросов   |
| st002012 | Регуляторы / муниципальные службы         | Соответствие законам и нормам (транспорт, санэпиднадзор)                          | Отчётность, хранение данных, соблюдение правил перевозки продуктов, безопасность данных      | Изменяющиеся нормативы, штрафы, необходимость дополнительных процедур для соответствия   |
| st002013 | Партнёры по логистике / fulfilment центры | Эффективная комплектация и передача заказов                                       | Интеграция/стандарты обмена заказами, SLA по выдаче, отслеживание статусов                   | Проблемы с синхронизацией, очередь на комплектацию, несоответствие ожиданий по SLA       |
| st002014 | Тестовые пользователи / ранние клиенты    | Возможность проверить сервис, получить выгоду от участия в пилоте                 | Сбалансированные фичи, каналы обратной связи, вознаграждение за участие                      | Неоптимальный UX, баги, недостаточная мотивация участвовать                              |
