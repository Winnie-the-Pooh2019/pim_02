
# Exercise 03 — Glossary (Task 2, prefix: del)

Дата добавления: PIM_02

Ключевые понятия и термины для задачи 2 (Доставка заказов). Таблица даёт компактный обзор: термин (EN) | термин (RU) | определение | примечания / связанные сущности.

| Термин                              | Определение                                                                                                                       | Примечания / Связанные сущности                             |
|-------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------|
| Заказ                               | Набор товаров, оформленный клиентом и подготовленный поставщиком для доставки; содержит адрес, товары, желаемое время и контакты. | Связанный с SKU, fulfillment, payment, tracking.            |
| Курьер                              | Исполнитель доставки, получающий задания и выполняющий развоз; работает через мобильное приложение.                               | Может иметь статус, рейтинг и баланс выплат.                |
| Диспетчер                           | Роль, контролирующая поток заказов и курьеров, перераспределяющая задачи и решающая исключения.                                   | Взаимодействует с dispatching algorithm и мониторингом.     |
| Пункт выдачи / Точка выдачи         | Локация, где клиент самостоятельно забирает заказ (магазин, постамат).                                                            | Альтернатива курьерской доставке.                           |
| Фулфилмент / Комплектация           | Процесс сборки и подготовки заказа на складе для отправки.                                                                        | Включает проверку, упаковку, маркировку.                    |
| Оператор ввода заказов              | Человек, нормализующий входящие заказы и вводящий их в систему.                                                                   | Часто обрабатывает исключения и корректирует данные.        |
| Мобильное приложение                | Интерфейс для курьеров: просмотр, бронирование и отметки статусов заказов.                                                        | Источник телеметрии и статусов.                             |
| API / Webhook / Интеграция          | Технические интерфейсы для передачи заказов, статусов и оплат между системами.                                                    | Включает order events, tracking updates, payment callbacks. |
| Провайдер платежей                  | Внешний сервис для обработки оплат и подтверждения транзакций.                                                                    | Интегрируется с accounting и order lifecycle.               |
| SLA (Service Level Agreement)       | Правила/соглашения по срокам комплектации и передачи заказов курьерам.                                                            | Используется для расчёта компенсаций и KPI.                 |
| Алгоритм диспетчеризации            | Логика распределения заказов между курьерами (радиус, приоритет, загрузка).                                                       | Влияет на ETA и стоимость.                                  |
| ETA / Estimated Time of Arrival     | Предполагаемое время прибытия курьера к клиенту.                                                                                  | Рассчитывается на основе трека и маршрута.                  |
| POD / Proof of Delivery             | Подтверждение вручения заказа клиенту (подпись, фото, код).                                                                       | Хранится как доказательство исполнения.                     |
| Интеграция с бухгалтерией           | Передача данных о заказах и оплатах во внешнюю бухгалтерскую систему.                                                             | Для расчёта выплат и учёта доходов.                         |
| Личный кабинет курьера              | Интерфейс, где отображается баланс, история заказов и настройки курьера.                                                          | Включает выплаты, штрафы и KPI.                             |
| Неявка / Срыв доставки              | Ситуация, когда клиент не принял заказ или доставка невозможна.                                                                   | Триггерит возврат или штрафы, требует обработки.            |
| Пропускная способность              | Показатель количества заказов, которые система/операция может обработать за единицу времени.                                      | Важно для масштабирования.                                  |
| Telemetry & Monitoring / Телеметрия | Набор метрик и логов для отслеживания состояния системы, задержек и поведения курьеров.                                           | Используется для алертинга и анализа производительности.    |
| SKU / Артикул                       | Единица учёта товара на складе.                                                                                                   | Связан с inventory и order lines.                           |
| Геокодинг                            | Преобразование адреса в координаты (lat/lon) и обратно для расчёта расстояний и маршрутов.                                         | Обычно внешний API (карты/геосервис).                       |
| Маршрутизация                        | Расчёт оптимального пути и времени в пути между точками (pickup → dropoff).                                                        | Используется для ETA и подсказок курьеру.                   |
| GPS‑трекинг                          | Передача координат курьера для построения трека и мониторинга выполнения.                                                          | Часто часть мобильного приложения курьера.                  |
| Push‑уведомление                      | Уведомление на устройство (мобильное/веб) через push‑провайдера.                                                                   | Альтернатива SMS/email, требует токенов устройств.          |
| RBAC (Role‑Based Access Control)     | Разграничение прав доступа на основе ролей (оператор/диспетчер/админ/курьер).                                                      | Важен для безопасности и соответствия требованиям.          |
| Аудит‑лог                             | Журнал действий пользователей и системных событий (кто/что/когда сделал).                                                          | Нужен для расследований и регуляторных проверок.            |
| Инцидент                              | Отклонение от нормального сценария доставки (опоздание, отказ, проблема с адресом/товаром), требующее обработки.                   | Триггерит эскалации и переназначения.                       |
| Идемпотентность                       | Свойство повторного запроса (например, создание заказа) не создавать дубликаты при повторах.                                       | Обычно достигается idempotency‑ключом.                      |
| Атомарная бронь                        | Бронирование заказа курьером как единая операция без «двойного назначения».                                                       | В контексте свободных заказов и конкуренции курьеров.       |
